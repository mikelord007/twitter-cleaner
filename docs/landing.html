Features

Everything you need to
wipe the slate clean.

A focused CLI with no cruft — just the controls you need to surgically clean your history.

🗄️
Full archive support
Point it at your downloaded Twitter archive and process your entire history — all the way back to your first tweet, not just the last 3,200.
parse tweets.js like.js
🖥️
Real browser, no API key
Uses Playwright with your installed Chrome — the same way you'd browse manually. Twitter sees a real user session, not a bot hitting API endpoints.
Playwright Chrome persistent session
🗑️
Five content types
Delete tweets, replies, quote tweets, undo retweets, and unlike posts — all from a single command. Mix and match with --type.
tweet reply quote retweet like
📅
Date range filtering
Target a specific time window with --before and --after. Items outside the range stay pending so a future run without filters can still pick them up.
--before --after YYYY-MM-DD
🧠
LLM content filter
Describe what to delete in plain English — "angry political tweets" or "self-promotional posts". Supports OpenAI, Anthropic, and OpenRouter (free tier available).
OpenAI Anthropic OpenRouter
🛡️
Stealth mode
Auto-pauses for 3–6 minutes every 50 deletions, hovers before clicking, and patches browser fingerprints — so Twitter doesn't clock the session as automation.
--stealth rate-limit safe
💾
Resume-safe progress
Every item is tracked in a local SQLite database. Kill the process, lose power, get rate-limited — pick up exactly where you left off next run.
SQLite status reset
👁️
Dry run mode
Pass --dry-run and the browser navigates to each item, highlights what would be clicked, and shows you the full count — without deleting a single thing.
--dry-run preview
🔭
Live scraper
No archive? The scrape command scrolls your live profile and loads found IDs directly into the database. Capped at ~3,200 by Twitter.
scrape live profile

How it works

Up and running
in three steps.

No tokens, no OAuth dance. Just your archive, a terminal, and a few minutes.

1
Install & configure
Clone the repo, install dependencies via pip, and drop your Twitter handle into a .env file. That's the only required credential.
2
Load your history
Download your Twitter archive from account settings and run twitter-cleaner parse. Tens of thousands of items load in seconds.
3
Preview, then delete
Always dry-run first to confirm what will be touched, then run delete for real. Progress auto-saves so you can stop and resume anytime.
4
Check status & retry
Run status to see counts by type. Any failures? reset --status failed re-queues them for another pass.
terminal
# 1. Install
❯ pip install -e .
❯ playwright install chrome

# 2. Configure .env
❯ cp .env.example .env
# add TWITTER_USERNAME=yourhandle
# Parse your downloaded archive
❯ twitter-cleaner parse --archive-dir ./data

Parsed 14,821 tweets/retweets/replies, 6,203 likes.
New records added: 21,024

# No archive? Scrape the live profile instead
❯ twitter-cleaner scrape
# Always preview first
❯ twitter-cleaner delete --dry-run

# Then delete for real
❯ twitter-cleaner delete

# Target specific types & dates
❯ twitter-cleaner delete --type like --before 2022-01-01
# Check progress
❯ twitter-cleaner status

like 412 5791 0 0

# Re-queue any failures
❯ twitter-cleaner reset --status failed

Smart filtering

Delete exactly what
you mean to delete.

Combine date ranges with LLM-powered content matching to surgically remove only the content you want gone.

📅
Date range
Use --before and --after to target a specific window. Items outside the range stay pending — run again without the flag to process them later.
twitter-cleaner delete
  --after 2020-01-01
  --before 2022-06-01
  --type tweet
🧠
LLM content filter
Describe what to delete in plain English. An LLM reads each tweet and decides if it matches before the action runs. Combine with --dry-run to preview selections.
twitter-cleaner delete
  --filter "angry political tweets"
  --llm-provider openai
  --dry-run
🔀
Combined filters
Stack date ranges and LLM filters together. The date filter runs first (fast, no API calls), then the LLM only evaluates the remaining items.
twitter-cleaner delete
  --before 2021-01-01
  --filter "cringe content"
  --llm-provider anthropic
⚡
LLM providers
Bring your own key for OpenAI or Anthropic, or use OpenRouter which has a free tier — so content filtering costs nothing if you use the right model.
--llm-provider openai
--llm-provider anthropic
--llm-provider openrouter # free tier
--llm-model MODEL_NAME # optional
