twtr-cleaner

Delete your entire Twitter/X history — tweets, retweets, replies, quotes, and likes — via browser automation. No API key needed.

Playwright-based No API key required Archive + live scrape LLM content filtering SQLite progress tracking Dry-run safe
📖

Overview

twtr-cleaner automates the deletion of your Twitter/X history by controlling a real browser with Playwright. It tracks progress in a local SQLite database so you can safely pause and resume at any time.

There are two ways to feed it your data: parsing a downloaded archive (full history) or live-scraping your profile (last ~3200 tweets). Once loaded, you can filter by date range or use an LLM to target specific content before deleting.

📦 Archive (recommended)

  • Full tweet history — all time
  • Works offline once parsed
  • Includes all types: tweets, replies, retweets, quotes, likes
  • Takes ~minutes to download from Twitter

🌐 Live Scrape

  • No archive download needed
  • Limited to last ~3200 tweets by Twitter
  • Useful for quick recent cleanup
  • Slower — requires active browser session
⚙️

Installation

1

Install (Python 3.11+)

pip install twtr-cleaner

Chromium is downloaded automatically the first time you run a command that needs the browser (~100 MB, one-time).

Method 1 — Parse Archive

The recommended approach for deleting your full history. Twitter lets you download your entire archive from Settings → Your account → Download an archive of your data. This can take 24–48 hours to prepare.

Once downloaded, extract the archive and point the tool at the folder containing tweets.js, like.js, etc.

Command

twtr-cleaner parse --archive-dir ./twitter-archive/data

Options

FlagDefaultDescription
--archive-dir PATH data Directory containing your archive data files (tweets.js, like.js, etc.)

Example

# Default — looks for archive data in ./data
twtr-cleaner parse

# Custom archive path
twtr-cleaner parse --archive-dir ~/Downloads/twitter-2024/data

After parsing, check how many records were loaded:

twtr-cleaner status
🌐

Method 2 — Live Scrape

Scrapes your live Twitter profile directly instead of using a downloaded archive. Useful if you don't have an archive or only want to clean up recent activity.

⚠️
Twitter limits profile visibility to the last ~3200 tweets. Use the archive method for full history.

Command

twtr-cleaner scrape

Options

FlagDefaultDescription
--tweets / --no-tweets on Scrape the tweets and replies tabs
--likes / --no-likes on Scrape the likes tab
--headless / --no-headless off Run the browser in headless mode (no visible window)

Examples

# Scrape both tweets and likes (default)
twtr-cleaner scrape

# Only scrape tweets, skip likes
twtr-cleaner scrape --no-likes

# Run headlessly in the background
twtr-cleaner scrape --headless
🗑️

Delete

Once your data is loaded (via parse or scrape), run delete. The browser will open — if you are not already logged in, sign in manually, then the tool will start processing items from the database.

twtr-cleaner delete [--type TYPE] [OPTIONS]
⚠️
Deletion is permanent and irreversible. Always do a --dry-run first.

Options

FlagDefaultDescription
--type TYPE all What to delete: tweets, replies, quotes, retweets, likes. Repeatable — pass multiple times. Omit to delete everything.
--dry-run off Navigate to each item and highlight what would be clicked, without actually deleting.
--headless / --no-headless off Run browser without a visible window
--stealth / --no-stealth on Pause for 3–6 minutes every 50 deletions to avoid Twitter rate-limiting. Pass --no-stealth to skip these breaks.
--min-delay FLOAT 3.0 Minimum seconds to wait between actions
--max-delay FLOAT 6.0 Maximum seconds to wait between actions
--before YYYY-MM-DD — Only delete posts published before this date
--after YYYY-MM-DD — Only delete posts published after this date
--filter TEXT — Natural language description of content to delete. Requires --llm-provider.
--llm-provider — openai, anthropic, or openrouter
--llm-model MODEL — Specific model name. Defaults to a cheap model per provider.
--llm-api-key KEY env var API key for the LLM provider. Falls back to env vars.

Examples

# Dry run — preview everything that would be deleted
twtr-cleaner delete --dry-run

# Delete everything
twtr-cleaner delete

# Delete only likes
twtr-cleaner delete --type likes

# Delete only replies, before 2026
twtr-cleaner delete --type replies --before 2026-01-01

# Delete tweets, replies and quotes — not retweets or likes
twtr-cleaner delete --type tweets --type replies --type quotes

# Delete everything, headlessly, with faster pacing
twtr-cleaner delete --headless --min-delay 1.5 --max-delay 3.0
📅

Date Filters

Use --before and --after to restrict deletion to a specific time window. Both flags accept dates in YYYY-MM-DD format.

💡
When using both flags, --after must be earlier than --before. The range is exclusive on both ends.

Examples

# Delete everything posted before 2022
twtr-cleaner delete all --before 2022-01-01

# Delete tweets from a specific year (2020)
twtr-cleaner delete tweets --after 2020-01-01 --before 2021-01-01

# Unlike all likes before 2023, dry run first
twtr-cleaner delete likes --before 2023-01-01 --dry-run
🤖

LLM Content Filters

Use the --filter flag to describe in plain English what kind of content to delete. An LLM reads each tweet and decides whether it matches your description before the tool deletes it.

This requires an API key from one of three supported providers:

🟢
OpenAI

GPT-4o, GPT-4o-mini, etc.

🟠
Anthropic

Claude Sonnet, Haiku, Opus

🔵
OpenRouter

Multi-provider. Has a free tier.

Examples

# Delete angry or political tweets using OpenAI
twtr-cleaner delete tweets \
  --filter "angry political tweets" \
  --llm-provider openai

# Use Anthropic Claude with a specific model
twtr-cleaner delete tweets \
  --filter "self-promotional posts about my startup" \
  --llm-provider anthropic \
  --llm-model claude-haiku-4-5-20251001

# Use OpenRouter (free tier available)
twtr-cleaner delete tweets \
  --filter "tweets mentioning competitors" \
  --llm-provider openrouter \
  --llm-model mistralai/mistral-7b-instruct

# Combine with a date range — old embarrassing tweets
twtr-cleaner delete tweets \
  --before 2020-01-01 \
  --filter "cringe or embarrassing content" \
  --llm-provider openai \
  --dry-run
💡
The API key can be passed via --llm-api-key or set in your .env file as OPENAI_API_KEY, ANTHROPIC_API_KEY, or OPENROUTER_API_KEY. No need to pass it on the command line.

Status

Check how many items are pending, done, failed, or skipped — broken down by type.

twtr-cleaner status

Outputs a table like:

┌──────────┬─────────┬──────┬────────┬─────────┐
│ Type     │ Pending │ Done │ Failed │ Skipped │
├──────────┼─────────┼──────┼────────┼─────────┤
│ tweet    │     120 │  340 │      5 │       0 │
│ reply    │      45 │  210 │      2 │       0 │
│ retweet  │      80 │  150 │      1 │       0 │
│ like     │     200 │  500 │      0 │      12 │
└──────────┴─────────┴──────┴────────┴─────────┘

Run this after parsing to confirm records were loaded, and after deletion runs to track progress.

🔁

Reset

Re-queue items for retry by resetting their status back to pending. Useful after transient failures or network errors.

twtr-cleaner reset [OPTIONS]

Options

FlagDefaultDescription
--type TYPE — Only reset items of this type: tweets, replies, retweets, quotes, or likes
--status STATUS failed Reset items with this status: failed, skipped, or done

Examples

# Re-queue all failed items (default)
twtr-cleaner reset

# Re-queue only failed tweets
twtr-cleaner reset --type tweets --status failed

# Re-queue skipped likes (e.g. to retry with different settings)
twtr-cleaner reset --type likes --status skipped

# Reset everything that was marked done (USE WITH CAUTION)
twtr-cleaner reset --status done

Troubleshooting

Common issues and how to fix them.

Login fails or gets stuck on a CAPTCHA / verification screen ▼

Login is fully manual — the tool opens a real Chrome window and waits for you to sign in yourself. Enter your password, complete any 2FA or CAPTCHA prompts as you normally would, and the tool will detect when you're done and continue automatically.

Make sure you are not using --headless on the first run, otherwise the browser window is invisible and you cannot log in.

Once you are logged in, the session is saved to .twitter_cleaner/chrome_profile/ and reused on all future runs — you should not need to log in again unless Twitter expires your session.

Many items are marked failed — timeouts on every tweet ▼

This usually means Twitter is rate-limiting or temporarily blocking requests. Increase the delay between actions:

twtr-cleaner delete tweets --min-delay 8 --max-delay 15

After the run, reset failed items and retry: twtr-cleaner reset

How do I retry failed items? ▼

Items that fail 3 times are no longer retried automatically. Reset them back to pending and run delete again:

twtr-cleaner reset          # resets all failed items
twtr-cleaner delete tweets   # retries them
Dry-run messed up my progress — items are now showing as done ▼

This was fixed in a recent update — dry-run no longer writes any status changes to the database. If you ran an older version, reset the affected items:

twtr-cleaner reset --status done

Caution: this also resets anything that was genuinely deleted. Only do this if the done count doesn't reflect real deletions.

Lots of items are showing as skipped unexpectedly ▼

Items are skipped when they return a 404 (already deleted), the tweet is unavailable, or they fall outside your date filter. Check your --before / --after flags.

To retry skipped items with different settings: twtr-cleaner reset --status skipped

parse loads 0 records — no tweets found in archive ▼

Make sure you're pointing at the right directory. The tool looks for tweets.js / tweets-part1.js etc. inside the path you provide. Your archive should have a data/ folder — use that:

twtr-cleaner parse --archive-dir ~/Downloads/twitter-archive/data
LLM filter is deleting everything (or nothing) ▼

Be specific in your description. Vague prompts like "bad tweets" tend to match everything. Try something like "tweets expressing strong political opinions" or "self-promotional posts about my side project".

Always combine with --dry-run to preview what the LLM selects before actually deleting:

twtr-cleaner delete tweets --filter "your description" --llm-provider openai --dry-run
Can I pause and resume a deletion run? ▼

Yes — just press Ctrl+C to stop. Progress is saved to the SQLite database after every item. The next time you run the same delete command it will pick up where it left off, skipping already-processed items.
